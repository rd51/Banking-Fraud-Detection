// PROBLEM STATEMENT

The gaps this prototype closes

GAP 01
CRITICAL
Rule-based monitoring generates 95%+ false positives
Legacy AML systems flag transactions using static thresholds — "flag any transfer above $10K from a high-risk country." The result: compliance teams spend 90%+ of their time clearing legitimate transactions while sophisticated layering schemes that stay below individual thresholds slip through undetected. Regulators now explicitly state that static rule sets are no longer tolerable — penalties for transaction monitoring failures rose 100% year-on-year in 2024.
GAP 02
CRITICAL
No temporal reasoning — each transaction scored in isolation
Money laundering is revealed by sequence, not by individual events: small deposits → rapid cross-account movement → currency conversion → withdrawal in a different jurisdiction. Rule-based and simple ML systems score each transaction in isolation, permanently blind to the trajectory. LSTM is specifically required to model the 90-day behavioural window that reveals layering patterns.
GAP 03
HIGH
KYC document fraud not integrated with transaction monitoring
Banks run document verification at onboarding and transaction monitoring separately — in isolated silos that never talk to each other. No deployed system fuses both: a CNN reading document-image anomalies (font inconsistency, metadata tampering in uploaded ID scans) with the LSTM tracking subsequent transaction patterns. This is precisely the seam sophisticated fraud rings exploit.
GAP 04
HIGH
Black-box model outputs can't generate SAR narratives
When a model flags an account, compliance officers must file a Suspicious Activity Report with a written, human-readable justification. Deep learning outputs (a fraud score) don't map to SAR narrative requirements. Either officers over-ride model flags — wasting the model — or file incomplete SARs, risking regulatory sanction. Agentic NLP-generated SAR drafts are the missing layer the industry has not yet standardised.
// MODEL DESIGN

Tri-modal deep learning architecture

BRANCH A · CNN
Document anomaly detector
Reads uploaded KYC image files — passports, utility bills, company docs. Extracts spatial features: font inconsistency, pixel artefacts, metadata tampering. MobileNet / EfficientNet backbone fine-tuned on forgery datasets. Stride 1–3 convolutions produce feature maps per filter.
output → document_anomaly_score [0–1]
BRANCH B · LSTM RNN
Transaction sequence encoder
Reads 90-day rolling transaction window: amount, merchant category code, channel, country, time-delta. Each transaction is one timestep. LSTM hidden state carries forward the account's behavioural fingerprint. MinMaxScaler normalises monetary values before input.
output → sequence_risk_vector [128-dim]
BRANCH C · DENSE (ANN)
Account profile baseline
Reads structured account metadata: account age, country risk tier (FATF list), PEP / sanctions flag, customer segment, historical SAR count, KYC tier, product type. MinMaxScaler + one-hot encoding. Dense layers extract latent risk context.
output → profile_risk_vector [64-dim]

// AGENTIC AI LAYER

What the agent actually does after the model predicts

🔀
Smart alert routing
When the fusion score crosses a threshold, the agent routes automatically: score >0.9 → instant freeze queue, 0.6–0.9 → senior analyst review with evidence packet, <0.6 → auto-clear with audit log. Eliminates >90% of manual triage.
agentic automation
📋
SAR narrative drafting
NLP keyword extraction (Day 1 topic) reads the flagged transaction sequence and CNN anomaly signals, extracts key entities — amounts, dates, geographies, merchant codes — and auto-drafts a SAR in regulatory format. Analyst reviews and signs off in one click.
NLP + keyword extraction
🔄
Continuous model retraining
Agent monitors precision / recall on new live cases. When performance drifts below the threshold, it triggers retraining on fresh labelled data — directly applying the Day 1 principle: "continuous training is required for the model." Keeps the model current with evolving fraud patterns.
continuous training
🛡️
Sanctions & PEP screening agent
On each new onboarding, agent checks customer name + entity against live OFAC, UN, and EU sanctions lists via MCP connector. Flags PEP status and adverse media hits. Feeds result directly into the Dense branch account profile before model inference.
MCP servers

// REAL-TIME AI LANDSCAPE

Where this sits in the 2025–26 AI world

LSTM is the fastest-growing architecture in financial fraud detection (2022–2024)
A systematic review of 5 years of DL fraud detection literature (arXiv 2025) confirms LSTM shows the most significant and sustained growth, culminating in a sharp increase from 2022 to 2024 — driven by the sequential nature of fraud datasets. Your project sits at the peak of this curve.
Explainability (XAI) is now a regulatory requirement, not a nice-to-have
The FATF has identified explainability gaps in AML AI as a critical weakness. The ScienceDirect hybrid DL-XAI paper (Jan 2026) shows that combining feature fusion with interpretability outputs is where the field is heading. Your agentic SAR-drafting layer directly addresses this gap.
Digital assets and fintechs are the fastest-growing enforcement target
Digital asset firms represent almost one quarter of the top 10 highest-value AML fines in 2025 (ComplyAdvantage). OKX paid $504M. Coinbase Europe was penalised for 176 billion EUR in unchecked transactions. A system like this prototype is exactly what these institutions lack.
EU AMLA (2027) will make European enforcement match US intensity
The EU's Anti-Money Laundering Authority begins direct supervision in 2027. Current North American / rest-of-world enforcement split (95%/5%) will narrow sharply from 2028. Banks and fintechs in EMEA — including UAE institutions with EU operations — are building now to be ready. This prototype can be pitched to any compliance team in that pipeline.
